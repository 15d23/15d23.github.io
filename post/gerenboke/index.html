使用“GitHub”或“Gitee码云”建立个人博客

2020-07-02

基于"GitHub"或"Gitee码云"结合Gridea
不花一分钱，建立无广告个人博客。

1. GitHub

  • 1-2、注册github（例：mygithub)

  • 1-2、创建仓库(repository)

需要带readme.md文件以方便之后访问网站内容。
仓库名称：myblog.github.io (必需是name.github.io)。

  • 1-3、访问

https://myblog.github.io 应该能看到readme.md页面。

2. Gridea

  • 2-1、Gridea简介

基于本地写博客同步至github的一个软件，非常好用。

  • 2-2、官方网址

官网 Github
下载地址：Download

  • 2-3、官方教程：

  • 2-4、进阶配置--留言

    • 注册Gtalk账号

3. Gitee码云

  • 3-1、简介

与github一样，是国内较好的代码仓库管理平台，也可以做静态博客使用。

  • 3-2、创建Gitee博客

    • 注册Gitee（例：mygitee)
    • 创建与账户同名仓库（mygitee)，这样可以使用mygitee.gitee.io做为网址了。参看码云Pages
    • 进入仓库，点击“服务--gitee pages”绑定手机后布署服务即可。
    • 访问mygitee.gitee.io应该就能看到网页了。

4. 布署Gridea到Gitee

  • 4-1、创建Gitee令牌

  • 4-2、将Gridea推送至Gitee

  • 4-3、修改Gridea配置文件适应Gitee

对应github，将配置改为gitee相对应的值后点击“保存”。
打开站点源文件路径下"config/setting.json"文件。
将“platform”的值改为“gitee”保存。这样生成的网站主域就是“gitee”了。
点击“同步”会失败，没关系。我们手动推送。

  • 4-5、推送

由于配置的时候没有gitee选项，所以生成的推送域名是github。
手动修改远端地址为gitee，强制推送即可。（有时间我再详细写）
* 想要一个仓库推送到多个远程仓库，请参看我的另一篇文章《Git杂记：本地仓库提交多个远端仓库》
